# فحص التأثير الإيجابي والسلبي (Impact Analysis)

**إجباري قبل أي تعديل على الكود.**
**يشمل: التأثير الإيجابي + التأثير السلبي المحتمل على جميع الجوانب.**

## الخطوات:

### 1. تحديد التغيير
- ما الملف/الدالة/السطور التي ستتغير؟
- ما المشكلة التي نحلها؟

### 2. تحليل التأثير
- ما الدوال/المتغيرات التي تعتمد على هذا الكود؟
- ما عناصر UI المرتبطة؟
- هل هناك حسابات أخرى تستخدم نفس المتغيرات؟

### 3. التأثيرات المتوقعة
**إيجابية:**
- (اكتب ماذا سيُصلح)

**سلبية محتملة:**
- (اكتب ما قد يتأثر سلباً)

### 4. خطة الاختبار
- كيف ستتأكد أن التغيير يعمل؟
- ما الأجزاء الأخرى التي يجب فحصها؟

---

## القالب للتقرير:

```
## تقرير فحص التأثير

**المشكلة:** [وصف المشكلة]

**الحل المقترح:** [وصف التغيير]

**الملفات المتأثرة:**
- [ ] ملف1.js (الدالة X)

**التأثيرات الإيجابية:**
- ✅ [ما سيُصلح]

**التأثيرات السلبية المحتملة:**
- ⚠️ [ما قد يتأثر]

**خطة الاختبار:**
- [ ] فحص كذا
- [ ] فحص كذا

**الحالة:** ⏳ بانتظار الموافقة
```

---

## القواعد:

1. **لا تغيير بدون تقرير** - كل تعديل يحتاج تقرير تأثير
2. **لا تنفيذ بدون موافقة** - انتظر موافقة المستخدم
3. **تغيير واحد في كل مرة** - لا تجمع تغييرات متعددة
4. **اختبر قبل الـ push** - تأكد أن الكود يعمل
